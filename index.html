1.) I answered the FIFA questions. My line graph answers the first question by showing from 2000-2004, the number of football games per each of those years. One can follow the line starting from 2000, and see how the number of football games per year have changed up til 2004. My graph answers the second question by showing the overall top 10 nations in FIFA by winning percentages, and it’s shown in descending order which helps readers quickly spot the top 10 nations by ranking in order. Furthermore, readers can quickly see the winning percentages of each nation next to the bars.

My graph answers the third question by showing the overall (top 15) nations most likely to win the next Fifa World Cup, by ranking the countries based on winning percentages or victory strengths. The data chosen was from the FIFA World Cup or Fifa World cup qualifiers in 2011-2018, which essentially are the last 2 World Cups. Readers can quickly see the winning percentages or victory strength of each nation next to the bars, in descending order, where the top bar or nation represents the nation most likely to win the next FIFA World Cup.

-d3 worked seamlessly with other web technologies such as HTML, CSS and SVG.

- d3 allowed me to manipulate Document Object Models (DOM) based on my data.

3.) 3 reasons d3 would not be the best tool for creating a visualization:

- Document Object Model manipulation can be inefficient and slow if there are a lot of inputs.

- Creating charts from d3 can be time-inefficient, in that there are other data visualizations libraries that have pre-generated charts made for me already that I can easily use.

- d3 involves using Javascript, which can be hard to learn and pick up, in contrast to using other data visualization library that may have a smaller learning curve.
